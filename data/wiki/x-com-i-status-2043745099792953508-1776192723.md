# X.com JavaScript Requirement Error

## Summary

This article documents a technical error encountered when accessing X.com (formerly Twitter) without [[JavaScript]] enabled in the browser. The error page provides information about browser requirements and troubleshooting steps for users experiencing this issue.

## Overview

X.com requires [[JavaScript]] to function properly. When a user attempts to access the platform with [[JavaScript]] disabled, they encounter an error message that prevents them from using the service. This is a common requirement for modern web applications that rely on client-side scripting for interactivity and functionality.

## Error Details

### JavaScript Disabled Detection

The platform automatically detects when [[JavaScript]] is not available in the user's browser and displays an error notification. The error message states:

> "We've detected that JavaScript is disabled in this browser. Please enable JavaScript or switch to a supported browser to continue using x.com."

### Browser Compatibility

X.com maintains a list of supported browsers that have been tested and verified to work with the platform. Users experiencing issues are directed to consult the Help Center for a complete list of compatible browsers.

## Troubleshooting

### Primary Solutions

1. **Enable JavaScript** - Users should enable [[JavaScript]] in their browser settings
2. **Switch Browsers** - Use a supported browser that is compatible with X.com
3. **Disable Privacy Extensions** - Some privacy-related extensions may interfere with X.com functionality and should be disabled
