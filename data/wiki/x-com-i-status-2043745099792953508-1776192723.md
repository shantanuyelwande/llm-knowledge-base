# X.com JavaScript Requirement Error

## Summary

This article documents a common technical error encountered when accessing X.com (formerly Twitter) with [[JavaScript]] disabled in the web browser. The error page provides guidance for users to enable JavaScript or switch to a supported browser to continue using the platform.

## Overview

X.com requires [[JavaScript]] to function properly. When a user attempts to access X.com with JavaScript disabled, the platform displays an error message preventing further navigation until JavaScript is enabled or a compatible browser is used.

## Error Details

### What Causes This Error

The error occurs when:
- [[JavaScript]] is disabled in the browser settings
- The user is accessing X.com through an unsupported browser
- Browser extensions related to privacy or security block JavaScript execution

### Error Message

Users encountering this issue see the following message:

> We've detected that JavaScript is disabled in this browser. Please enable JavaScript or switch to a supported browser to continue using x.com.

## Solutions

### Enabling JavaScript

Users can resolve this issue by:
1. Enabling JavaScript in their browser settings
2. Consulting the [X.com Help Center](https://help.x.com/using-x/x-supported-browsers) for browser-specific instructions
3. Disabling privacy-related extensions that may interfere with JavaScript execution

### Browser Compatibility

X.com maintains a list of [[supported browsers]] in their Help Center. Users experiencing persistent issues should verify their browser is on the supported list.

### Troubleshooting Privacy Extensions

The error page notes that some [[privacy]] related extensions may cause issues on X.com. Users should:
- Temporarily disable privacy extensions
- Whitelist X.com in extension settings
- Try again after making changes
