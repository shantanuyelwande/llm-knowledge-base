# X.com JavaScript Requirement Error

## Summary

This article documents a technical error encountered when accessing X.com (formerly Twitter) without [[JavaScript]] enabled in the browser. The error message indicates that JavaScript is a mandatory requirement for using the platform's web interface.

## Overview

X.com requires [[JavaScript]] to function properly. When a user attempts to access the platform with JavaScript disabled, they encounter an error page that prevents further interaction with the site. This is a common practice among modern web applications that rely heavily on client-side scripting for dynamic functionality.

## Error Message and Content

The error page displays the following key message:

> "We've detected that JavaScript is disabled in this browser. Please enable JavaScript or switch to a supported browser to continue using x.com."

### Recommended Actions

The error page suggests two solutions:

1. **Enable JavaScript** - Users can enable JavaScript in their browser settings
2. **Switch to a Supported Browser** - Users can migrate to a browser that supports JavaScript and is officially supported by X.com

Users are directed to the [Help Center](https://help.x.com/using-x/x-supported-browsers) for a list of supported browsers.

## Technical Context

### Browser Requirements

X.com's reliance on JavaScript reflects modern web application architecture, where client-side scripting handles:

- [[API communication]]

The error page includes a notice regarding privacy-related browser extensions:

> "Some privacy related extensions may cause issues on x.com. Please disable them and try again."

This suggests that certain [[privacy extensions]] may interfere with the platform's JavaScript execution or functionality.
