# X.com JavaScript Requirement Error

## Summary

This article documents a technical error encountered when accessing X.com (formerly Twitter) with [[JavaScript]] disabled in a web browser. The error page provides guidance on enabling JavaScript and troubleshooting browser compatibility issues.

## Overview

X.com requires [[JavaScript]] to function properly. When a user attempts to access the platform with JavaScript disabled, they encounter an error message that prevents access to the service. This is a common requirement for modern web applications that rely on client-side scripting for dynamic functionality.

## Error Details

### JavaScript Disabled Error

When [[JavaScript]] is not enabled in a user's browser, X.com displays a blocking error message that:

- Prevents access to the main platform
- Provides instructions for enabling [[JavaScript]]
- Directs users to supported browser information
- Offers troubleshooting options

### Error Message

The primary error notification states:

> We've detected that JavaScript is disabled in this browser. Please enable JavaScript or switch to a supported browser to continue using x.com.

## Troubleshooting Solutions

### Enable JavaScript

Users can resolve this issue by:

1. Enabling [[JavaScript]] in their browser settings
2. Checking browser security preferences
3. Disabling conflicting browser extensions

### Browser Compatibility

X.com maintains a list of [[Supported Browsers|supported browsers]] in their Help Center. Users experiencing persistent issues should verify their browser is on the compatibility list.

### Privacy Extensions

Some [[Browser Extension|privacy-related browser extensions]] may interfere with X.com functionality. The error page specifically notes:

> Some privacy related extensions may cause issues on x.com. Please disable them and try again.

Users should temporarily disable privacy extensions to test if they are causing the problem.
