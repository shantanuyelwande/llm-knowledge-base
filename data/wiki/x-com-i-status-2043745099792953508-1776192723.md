# X.com JavaScript Requirement Error

## Summary

This article documents a technical error encountered when accessing X.com (formerly Twitter) without [[JavaScript]] enabled in the browser. The error message indicates that JavaScript is a mandatory requirement for the platform to function properly.

## Overview

When attempting to access X.com with [[JavaScript]] disabled, users encounter a blocking error page that prevents further navigation. This represents a significant [[web accessibility]] concern, as it requires users to enable potentially resource-intensive scripting to use the platform.

## Error Details

### Error Message

The primary error message states:

> "We've detected that JavaScript is disabled in this browser. Please enable JavaScript or switch to a supported browser to continue using x.com."

### Cause

The error occurs due to:
- [[JavaScript]] being disabled in the user's browser settings
- Use of an unsupported or outdated browser
- Browser extensions that block [[JavaScript]] execution

### Potential Solutions

Users encountering this error are directed to:

1. **Enable JavaScript** - Activate [[JavaScript]] support in browser preferences
2. **Switch Browsers** - Use a supported browser from X.com's approved list
3. **Disable Extensions** - Remove or disable privacy-related browser extensions that may interfere with [[JavaScript]] execution
4. **Consult Help Resources** - Visit the X.com Help Center for browser compatibility information

## Technical Implications

### Browser Compatibility

X.com requires users to access the platform through [[supported browsers]] that have full [[JavaScript]] support. The platform does not provide graceful degradation for users with [[JavaScript]] disabled.

### Privacy Extension Conflicts

The error message specifically notes that privacy-related extensions may cause issues, suggesting:
- Potential conflicts between privacy tools and X.com's [[JavaScript]] requirements
- Possible tracking or analytics dependencies within the platform's [[JavaScript]] code
