# Full Stack Agentic AI

## Summary

Full Stack Agentic AI refers to comprehensive systems that build autonomous [[AI agents]] capable of operating independently over extended periods. These systems combine [[Large Language Models]] (LLMs) with augmentations such as [[retrieval]], [[tools]], and [[memory]] to accomplish complex tasks with minimal human intervention.

---

## Overview

An **Agentic AI System** is an autonomous system designed to operate independently over extended periods, using various tools and workflows to accomplish complex tasks. Unlike traditional software systems, agentic systems can dynamically direct their own processes and adapt their approach based on the task at hand.

## Key Distinctions

### Workflows vs. Agents

The field of Agentic AI Systems can be categorized into two primary approaches:

**Workflows**
- Systems where [[LLMs]] and [[tools]] are orchestrated through predefined paths
- Follow predetermined sequences and decision trees
- Suitable for well-defined, structured processes

**Agents**
- Systems where LLMs dynamically direct their own processes and tool usage
- Maintain autonomous control over how they accomplish tasks
- Adapt their approach based on context and feedback
- Enable more flexible problem-solving

## Core Components

Building effective Agentic AI Systems requires establishing foundational components:

### Basic Building Blocks

The fundamental building block of Agentic AI Systems is an **LLM enhanced with augmentations**, including:

- **[[Retrieval]]** - Accessing external information sources and databases
- **[[Tools]]** - Integration with external services, APIs, and functions
- **[[Memory]]** - Retaining and recalling information across interactions

### Implementation Considerations

According to Anthropic's recommendations, successful implementation focuses on two key aspects:

1. **Use Case Tailoring** - Customizing capabilities to specific application domains and requirements
2. **Interface Design** - Providing an easy, well-documented interface for the LLM to interact with augmentations

### Platform Approaches

**Built-in Platforms**
- Powerful LLM platforms come with integrated retrieval, tools, and memory capabilities
- Streamlined implementation with pre-configured components

**API-Based Approaches**
- When using APIs, developers must manually connect the LLM with:
  - Relevant tools and functions
  - Memory systems and databases
  - Information retrieval mechanisms
- LLMs can generate their own search queries, select appropriate tools, and determine what information to retain
