# X.com JavaScript Requirement Error

## Summary

This article documents a browser compatibility error encountered when accessing X.com (formerly Twitter) with JavaScript disabled. The error prevents users from accessing the platform and provides guidance on enabling JavaScript or switching to a supported browser.

## Overview

When attempting to access X.com without JavaScript enabled, users encounter a blocking error page that prevents further navigation. This is a technical requirement of the X.com platform, which relies on [[JavaScript]] as a core dependency for its functionality.

## Error Details

### Error Message

The primary error message states:

> "We've detected that JavaScript is disabled in this browser. Please enable JavaScript or switch to a supported browser to continue using x.com."

### Error Causes

The X.com platform cannot function properly without JavaScript enabled. Users may encounter this error due to:

- JavaScript being manually disabled in browser settings
- Browser extensions blocking JavaScript execution
- Unsupported or outdated browser versions
- Privacy-focused browser configurations

## Solutions

### Enable JavaScript

Users should enable JavaScript through their browser settings:

1. Access browser preferences or settings
2. Navigate to privacy and security options
3. Locate JavaScript settings
4. Enable JavaScript execution

### Switch Browsers

If JavaScript cannot be enabled, users should switch to a [[Web Browser|supported browser]]. X.com maintains a list of compatible browsers in their Help Center.

### Disable Privacy Extensions

Certain [[Browser Extension|privacy-related extensions]] may interfere with X.com functionality. The platform recommends:

- Disabling privacy extensions temporarily
- Testing X.com access without extensions
- Re-enabling extensions selectively to identify conflicts
